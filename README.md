# Arch Linux Post-Install Setup

Automated post-install configuration for Arch Linux. Sets up a full Hyprland Wayland desktop with development tools, gaming, virtualization, and power management utilities. All configuration is prompted interactively — no files to edit.

## Quick Start

### One-liner (recommended)

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/probird5/arch-install/main/bootstrap.sh)
```

This clones the repo and launches the installer. All configuration is prompted interactively.

### Manual

```bash
# 1. Clone this repo
git clone https://github.com/probird5/arch-install.git /tmp/arch-install

# 2. Run the root phase (prompts for all config)
sudo /tmp/arch-install/install.sh

# 3. Reboot
reboot

# 4. Log in as your user and run the user phase
/tmp/arch-install/post-install.sh
```

## Interactive Configuration

The installer prompts for everything up front before making any changes. You review a summary and confirm before it proceeds.

| Prompt | Default | Description |
|---|---|---|
| Username | `probird5` | System user account to create |
| Hostname | `archlinux` | Machine hostname |
| Timezone | `America/Toronto` | System timezone (validated against `/usr/share/zoneinfo`) |
| Locale | `en_CA.UTF-8` | System locale |
| Keymap | `us` | Console keymap |
| GPU driver | `amd` | Options: `amd`, `nvidia`, `intel`, `none` |
| Hyprland desktop | Yes | Hyprland, Waybar, Thunar, etc. |
| Development tools | Yes | Neovim, Go, Rust, Python, Node.js, etc. |
| Docker | Yes | Docker and docker-compose |
| Virtualization | Yes | QEMU/KVM, virt-manager, libvirt |
| Gaming | Yes | Steam, Wine, Lutris, Gamescope |
| CachyOS repositories | No | v3 optimized packages, gaming meta, proton |
| AUR helper | `paru` | Options: `paru`, `yay` |
| Dotfiles repo | `https://github.com/probird5/dotfiles.git` | Git URL for GNU Stow dotfiles |
| Swap size | `8` GB | Swapfile size (0 to skip), with auto-detected RAM shown |

After answering, a full summary is displayed:

```
=== Configuration Summary ===

  Username:        probird5
  Hostname:        archlinux
  GPU:             amd
  Hyprland:        true
  Docker:          true
  ...

  Proceed with this configuration? [Y/n]
```

The configuration is saved to `user-config.sh` so `post-install.sh` can pick it up after reboot.

## Installation Phases

### Phase 1: Root Setup (`install.sh`)

Run as root on a freshly installed base Arch system. Performs the following in order:

1. **Pre-flight checks** — verifies root access and internet
2. **Interactive configuration** — all prompts gathered up front with confirmation
3. **Pacman configuration** — parallel downloads (10), color, multilib
4. **CachyOS repositories** — optional v3 optimized packages (if selected)
5. **Gaming configuration** — CachyOS gaming meta or standard packages (if selected)
6. **System configuration** — timezone, locale, keymap, hostname
7. **Package installation** — all packages assembled from config based on selections
8. **GRUB bootloader** — EFI install with os-prober for dual-boot
9. **NVIDIA configuration** — nouveau blacklist, DRM modeset, initramfs, GRUB params (if nvidia selected)
10. **User creation** — user account, sudo, groups (docker/libvirt added dynamically)
11. **greetd login manager** — tuigreet with session selection (F3 to switch)
12. **Steam Gamescope session** — couch gaming session entry (if CachyOS gaming enabled)
13. **Battery saver script** — toggle script at `/usr/local/bin/battery-saver`
14. **SSH configuration** — password auth, user-only, no root login
15. **Firewall** — firewalld with deferred SSH rule
16. **Swap file** — btrfs-native with dd fallback
17. **Services** — NetworkManager, Bluetooth, SSH, Docker, libvirtd, greetd, etc.
18. **Libvirt default network** — auto-starts the default NAT network for virt-manager
19. **Rust toolchain** — stable toolchain via rustup
20. **Post-install prep** — sets ownership and permissions on scripts

### Phase 2: User Setup (`post-install.sh`)

Run as your normal user after rebooting. Handles:

1. **AUR helper** — builds and installs paru (or yay) from source
2. **AUR packages** — installs any packages listed in `AUR_PACKAGES`
3. **Git** — global user/email configuration
4. **Dotfiles** — clones repo and stows all configs via GNU Stow
5. **Tmux** — TPM install, `.tmux.conf`, plugin installation
6. **Shell** — sets zsh as default, appends plugin sources to stowed `.zshrc`
7. **Theming** — GTK 2/3/4, Qt, cursor themes (Dracula + Nordzy)
8. **Environment variables** — Wayland, editor, cursor, Steam compat paths
9. **XDG directories** — standard user directories
10. **Flatpak** — adds Flathub remote
11. **Firewall rules** — enables SSH in firewalld
12. **PipeWire** — enables user-mode audio services
13. **Rust verification** — ensures toolchain is ready

## NVIDIA Driver Setup

When `nvidia` is selected as the GPU driver, the installer performs a full proprietary driver setup:

| Step | Detail |
|---|---|
| Packages | `nvidia-dkms`, `nvidia-utils`, `lib32-nvidia-utils`, `nvidia-settings`, `nvidia-hook`, `egl-wayland`, `libva-nvidia-driver` |
| Blacklist nouveau | `/etc/modprobe.d/nouveau-blacklist.conf` |
| DRM kernel modesetting | `/etc/modprobe.d/nvidia.conf` — `modeset=1`, `fbdev=1`, `PreserveVideoMemoryAllocations=1` |
| Initramfs | Adds `nvidia nvidia_modeset nvidia_uvm nvidia_drm` to `MODULES=()` in mkinitcpio, regenerates |
| GRUB parameters | Appends `nvidia_drm.modeset=1 nvidia_drm.fbdev=1` to `GRUB_CMDLINE_LINUX_DEFAULT` |
| Suspend/resume | Enables `nvidia-suspend`, `nvidia-hibernate`, `nvidia-resume` systemd services |

The `nvidia-dkms` package is used instead of `nvidia` so the driver works across all kernel variants (linux, linux-lts, linux-zen, etc.). The `nvidia-hook` pacman hook auto-rebuilds the module on kernel updates.

## CachyOS Repositories

When enabled, the script:

1. Detects your CPU instruction set level (x86-64-v3 or v4)
2. Imports the CachyOS GPG signing key
3. Creates mirrorlists for both the main and v3 repos
4. Adds `[cachyos]`, `[cachyos-v3]`, `[cachyos-core-v3]`, and `[cachyos-extra-v3]` repos before `[core]` in `pacman.conf`
5. Installs the CachyOS forked pacman (required for v3 architecture support)

This provides x86-64-v3 optimized rebuilds of Arch packages and access to CachyOS-specific packages (gaming meta, topgrade, proton-cachyos, wine-cachyos, etc.). The CachyOS pacman is a minimal fork that adds v3/v4 architecture recognition — without it, standard pacman rejects v3 packages.

> **Note:** When CachyOS repos are enabled, standard mesa packages (`mesa`, `lib32-mesa`, `vulkan-radeon`, etc.) are automatically excluded from the install list since CachyOS provides optimized `mesa-git` replacements that conflict with the official packages.

## Gaming

### CachyOS Gaming Packages

When both CachyOS repos and gaming are enabled, the following are installed:

| Package | Description |
|---|---|
| `cachyos-gaming-meta` | Meta package with gaming libraries and Proton-CachyOS |
| `cachyos-gaming-applications` | Steam, Heroic, Lutris, Gamescope, Goverlay, MangoHud |
| `proton-cachyos-slr` | Proton-CachyOS Steam Linux Runtime variant |
| `wine-cachyos` | CachyOS optimized Wine |
| `umu-launcher` | Required for running Proton-CachyOS in Lutris and Heroic |

### Standard Gaming Packages (without CachyOS)

If CachyOS repos are not enabled, the following standard Arch packages are installed instead:

`steam`, `lutris`, `gamescope`, `gamemode`, `lib32-gamemode`, `wine`, `wine-mono`, `wine-gecko`, `winetricks`, `lib32-gnutls`, `lib32-sdl2`

### Steam Big Picture Session

When CachyOS gaming is enabled, a **Steam Big Picture** session is registered as a Wayland session option in the tuigreet login screen. Press **F3** at the login screen to switch between sessions.

**Available sessions:**

| Session | Description |
|---|---|
| Hyprland | Default tiling Wayland compositor desktop |
| Steam Big Picture | Gamescope + Steam GamepadUI (couch gaming mode) |

**What the Steam session does before launching:**

1. Stops all running Docker containers
2. Shuts down all running libvirt VMs
3. Launches `gamescope -e -- steam -gamepadui -steamdeck`

The wrapper script is installed at `/usr/local/bin/steam-session`. It gracefully handles cases where Docker or libvirt are not installed or no containers/VMs are running.

**Controller support:** Bluetooth is enabled system-wide. Pair your controller once from Hyprland (via blueman or `bluetoothctl`), and it will auto-reconnect when powered on in the Steam session.

## Battery Saver Mode

A toggle script is installed at `/usr/local/bin/battery-saver`. Run it once to enable battery saver, run it again to disable it.

**Bind it to a key in your Hyprland config:**

```
bind = SUPER, F12, exec, battery-saver
```

### What it does

| | Enable | Disable |
|---|---|---|
| Power profile | `power-saver` | `balanced` |
| Screen brightness | 20% | 60% |
| Docker | stopped | started |
| libvirtd | stopped | started |

The script tracks state via `~/.cache/battery-saver-active` and sends a desktop notification on each toggle. Docker and libvirt operations use `sudo systemctl` and are skipped if the respective tools aren't installed.

## Dotfiles

Dotfiles are managed via [GNU Stow](https://www.gnu.org/software/stow/) and deployed during `post-install.sh`. The script clones the configured repo and stows all packages into `~/.config/` and `~/`.

Stowed packages: `alacritty`, `ghostty`, `hypr`, `i3`, `librewolf`, `nvim`, `picom`, `rofi`, `starship`, `waybar`, `wezterm`, `wlogout`, `zsh`

> **Note:** `tmux` is excluded from stow — it is managed by `tmux-setup.sh` which writes `.tmux.conf` directly and installs TPM plugins.
